
### **Haib Tailor Web App Development Prompt**

#### **1. Overview**

Build a **custom-tailoring** platform called **Haib Tailor** where users (customers) can order tailor-made clothes, interact with tailors, track orders, and manage payments. Tailors can register, create profiles, manage orders, and communicate with customers.

#### **2. Technology Stack**

* **Backend:** Node.js with Express.js
* **Frontend:** HTML, CSS (black and white theme), JavaScript (vanilla or React.js)
* **Database:** MongoDB or MySQL
* **Authentication:** JWT-based authentication (JSON Web Token) or OAuth
* **Payment Gateway:** Stripe or PayPal
* **Geolocation:** MapLibre/OpenStreetMap integration for location-based tailor search
* **File Storage:** AWS S3 (for images like fabrics or profiles)
* **CI/CD:** GitHub Actions for deployment

---

### **3. Functional Features Breakdown**

#### **Customer Features**

1. **User Authentication**

   * **Sign Up:** Customers should sign up with email, phone number, and password.
   * **Login:** After registration, users should log in using their email and password or via social media.
   * **Forgot Password:** Customers should be able to reset their password through an email-based reset link.
   * **JWT Tokens:** Use JWT tokens for secure sessions.

2. **Customer Dashboard**

   * Display customer profile info (name, email, profile picture, address).
   * **Order History:** View past orders with status updates.
   * **Measurement Records:** Store and update body measurements for future tailoring.
   * **Saved Tailors:** Ability to save and view favorite tailors.
   * **Wallet/Payments:** View available balance, add funds, and track payment history.

3. **Tailor Search**

   * Search for tailors using **MapLibre/OpenStreetMap** integration.
   * Filter results by **services**, **location**, and **rating**.
   * View tailor details including their specializations, pricing, and customer reviews.

4. **Order Management**

   * **Place an Order:** Customers can select tailoring services (suits, dresses, alterations) and submit measurements.
   * **Order Tracking:** Track the status of orders (e.g., processing, completed, delivered).
   * **Reviews & Ratings:** After receiving the order, customers can rate and review the tailor.

5. **Order Payment**

   * **Secure Payments:** Integrate **Stripe** or **PayPal** to securely handle payments.
   * **Order Invoice:** Generate and send a digital invoice to the customer upon payment.
   * **Refunds:** Admin functionality for handling refunds and cancellations.

---

#### **Tailor Features**

1. **Tailor Registration**

   * **Sign Up:** Tailors register with email, phone number, and business details.
   * **Profile Management:** Tailors can fill out their profile with shop location, services offered, business hours, and pricing.

2. **Dashboard**

   * **Order Management:** View all orders, their status (in-progress, completed), and customer details.
   * **Payment Monitoring:** Track payments for completed orders and manage commissions.
   * **Order Notifications:** Real-time notifications when a new order is placed or a customer updates an order.

3. **Measurement Management**

   * **Access Customer Measurements:** View customers' saved measurements when they place an order.
   * **Adjustments:** Tailors can request clarification or adjustments if measurements are unclear.

4. **Customer Communication**

   * **Messaging:** In-app messaging system to discuss orders, ask for more details, or send updates to customers.
   * **File Sharing:** Share fabric samples, design images, and other media through the app.

5. **Ratings & Reviews**

   * **Customer Feedback:** Tailors can view reviews from customers and respond to feedback.
   * **Reputation Management:** Tailors can improve their ratings by maintaining good service standards and responding to reviews.

---

#### **Admin Features**

1. **Admin Dashboard**

   * **User & Tailor Management:** Admins can view all users (customers and tailors), deactivate accounts, or manage permissions.
   * **Order Oversight:** View all active and completed orders, with options to intervene if necessary (e.g., dispute resolution, refund processing).
   * **Financial Analytics:** Track overall revenue, commissions, and order volumes.

2. **Content Management System (CMS)**

   * **Homepage:** Admin can manage website content such as featured tailors, promotional banners, and announcements.
   * **Tailor Approval:** Admins review and approve new tailors before they are added to the platform.
   * **Review Moderation:** Admin can remove inappropriate or false reviews submitted by customers.

3. **Reporting & Analytics**

   * **Revenue Reports:** Admins can generate monthly, weekly, or custom reports of platform earnings, tailor commissions, etc.
   * **Customer Behavior Analytics:** Track active users, returning customers, and their spending patterns.
   * **Tailor Performance:** Review top-rated tailors, average response times, and customer satisfaction scores.

---

### **4. UI/UX Design (Black and White Theme)**

#### **Aesthetic**

* **Color Scheme:** The primary colors will be **black and white** with shades of gray. Use white backgrounds with black text for easy readability. Use light gray borders for input fields, buttons, and cards.
* **Typography:** Use **serif fonts** (for elegance and readability), such as **Merriweather** for headings, and **Roboto** for body text. Ensure proper contrast for accessibility.
* **Button Styling:** Simple black buttons with white text. On hover, change the button to white with black text and add a subtle shadow effect.
* **Forms and Inputs:** Clean, minimal input fields with a black border on focus.

#### **Customer Pages**

* **Homepage:** A simple, clean landing page featuring a **search bar** for finding tailors, and **call-to-action buttons** (e.g., “Sign Up,” “Find Tailors”).
* **Tailor Search Results:** Tailors will be displayed in a grid with a brief profile snippet, including ratings, services, and location.
* **Order Form:** An easy-to-navigate form where customers input their measurements, choose fabrics, and confirm their order details.
* **Order Status Page:** Show progress in clear steps (e.g., “Order Placed,” “In Progress,” “Completed”) with estimated delivery dates.

#### **Tailor Pages**

* **Profile Setup:** Simple, straightforward form for entering business details, adding services, and setting pricing.
* **Order Management:** A list view of all active orders with status updates, quick links to customer messages, and the ability to mark orders as completed.
* **Reviews Section:** Display reviews on their profile, showing customer feedback with the option to reply.

#### **Admin Pages**

* **User & Tailor Overview:** Clean tables with search and filter options for easy management of customers and tailors.
* **Order Dashboard:** Visualize all orders with color-coded statuses for quick action.
* **Reports Page:** Present key metrics in a simple chart format (line or bar graphs) showing revenue trends, order volumes, etc.

---

### **5. Development Phases**

#### **Phase 1: Setup**

* **Initialize Node.js Project:** Set up the **Node.js** environment with **Express.js** for routing and backend APIs.
* **Database Setup:** Set up **MongoDB** (or MySQL) for storing customer and order data.
* **Authentication System:** Implement **JWT-based authentication** for secure login, sign-up, and session management.

#### **Phase 2: Backend API Development**

* **CRUD Operations for Users:** Develop APIs for managing customers, tailors, and orders (Create, Read, Update, Delete).
* **Order Management:** Develop APIs to place, update, track, and review orders.
* **Payment Integration:** Set up **Stripe/PayPal** for processing payments and handling refunds.
* **Geolocation API:** Implement MapLibre/OpenStreetMap for tailor search by location.

#### **Phase 3: Frontend Development**

* **Create Customer Dashboard:** Design and build out the UI for customer registration, login, and the dashboard with all required functionalities.
* **Design Tailor Profiles:** Implement tailor profile pages with their details, services, and ratings.
* **Admin Dashboard:** Develop the admin interface to manage users, orders, payments, and generate reports.

#### **Phase 4: Testing & Debugging**

* **Unit Testing:** Write tests for all critical backend functionalities (e.g., order creation, user authentication).
* **UI Testing:** Test all UI components for responsiveness, functionality, and usability.
* **Payment Gateway Testing:** Test payment workflows to ensure smooth transactions.

#### **Phase 5: Deployment**

* **CI/CD Setup:** Set up **GitHub Actions** for continuous integration and deployment.
* **Cloud Hosting:** Deploy the app on a platform like **Heroku** or **AWS** for scalability.

---

### **6. Final Notes**

* **Security:** Ensure data privacy and security by encrypting sensitive information (passwords, payment details).
* **Performance:** Optimize API calls, reduce load times, and ensure the system can handle traffic spikes.
* **Mobile-Friendly:** Ensure the app is responsive and works seamlessly on mobile devices.
